## Project Overview

This is a course website for Sta 663 (Statistical Computing and Computation) at Duke University. The website is built using Hugo (a static site generator) combined with Quarto for rendering lecture slides. The site includes lecture slides, a schedule, syllabus, and course information.

## Architecture

### Build System

The project uses a hybrid build system combining Hugo, Quarto, and Make:

1. **Quarto** (v1.9.12+) renders `.qmd` files to HTML slides using RevealJS
2. **Hugo** (v0.154.5+) generates the static website from layouts, data, and rendered content
3. **Make** orchestrates the build pipeline and manages dependencies
4. **renderthis** R package converts HTML slides to PDF format

### Directory Structure

- `static/slides/` - Source Quarto `.qmd` files for lecture slides
- `docs/` - Generated static site (published to GitHub Pages, do not edit manually)
- `layouts/` - Hugo HTML templates that define the site structure
  - `layouts/index.html` - Main page layout
  - `layouts/partials/` - Reusable template components (schedule, syllabus, menu, etc.)
- `data/` - YAML data files (e.g., `schedule.yaml` defines course schedule)
- `static/css/` - Custom CSS styles
- `config.yaml` - Hugo site configuration and metadata

### Content Flow

1. Lecture slides are written as Quarto `.qmd` files in `static/slides/`
2. Each `.qmd` file has YAML frontmatter with RevealJS settings and uses `slides.scss` theme
3. Slides can contain Python, R, or other executable code blocks
4. The schedule in `data/schedule.yaml` references slides by name (without extension)
5. Hugo templates render the schedule and link to the generated HTML/PDF slides

## Common Commands

### Building the Site

```bash
make build
```
Renders all Quarto slides to HTML and PDF, then runs Hugo to generate the complete site in `docs/`.

```bash
make html
```
Renders only the HTML slides (faster for development).

```bash
make pdf
```
Renders PDF versions of all slides using the renderthis R package.

```bash
make all
```
Renders both HTML and PDF slides, then builds the site.

```bash
make open
```
Builds the site and opens `docs/index.html` in your browser.

### Development Workflow

```bash
make clean
```
Removes all generated files (`docs/`, `static/slides/*.html`, `static/slides/*.pdf`).

```bash
make push
```
Builds the site and pushes to Git with a generic commit message "Make update".

### Rendering Individual Slides

```bash
quarto render static/slides/LecXX.qmd
```
Renders a single lecture slide file to HTML.

```bash
Rscript -e "renderthis::to_pdf('static/slides/LecXX.html')"
```
Converts a rendered HTML slide to PDF.

## Key Technical Details

### Slide Format

Lecture slides use Quarto with RevealJS format. Each `.qmd` file should have:
- YAML frontmatter specifying `format: revealjs` with appropriate theme and settings
- `self-contained: true` to embed all dependencies in a single HTML file
- Executable code blocks (Python, R) that run during rendering

### Schedule Management

The course schedule is defined in `data/schedule.yaml` with:
- `startdate` - First day of the semester
- `schedule` array with entries for each class session
  - `week` - Week number (0-indexed)
  - `day` - Day of week as number (1=Mon, 2=Tue, etc.)
  - `lecture` - Lecture title
  - `slides` - Slide filename (without extension, e.g., "Lec02")
  - `readings` - Array of reading links

Hugo templates in `layouts/partials/schedule.html` automatically calculate dates and generate links to slides.

### Deployment

The site is deployed to GitHub Pages from the `docs/` directory. The GitHub Actions workflow (`.github/workflows/static.yml`) automatically deploys the `docs/` directory on pushes to main.

## Important Notes

- **NEVER read, review, edit, or touch any files under `docs/`** - this entire directory is auto-generated by Hugo via the Makefile. If asked to review or edit a file in `docs/`, refuse and explain that it is a generated file that should only be modified by editing the source files and rebuilding via `make`
- The `static/slides/prev/` directory contains old versions of slides and is excluded from the published site
- Both `docs/slides/` and `static/slides/` contain `.Rproj.user` directories that should be ignored
- Slide rendering may take time as it executes all code blocks in the `.qmd` files
